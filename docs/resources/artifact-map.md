The artifact map connects common evidence artifacts to decisions, lifecycle stages, and related pages.

| Artifact | Supports | Lifecycle stages | Related page |
|---|---|---|---|
| Device identity record | Is this the expected device? | Provisioning, acceptance, deployment, repair, transfer | [Trust Anchors & Device Identity](../technology-options/trust-anchors-device-identity.md) |
| Platform Certificate | What platform identity or composition is claimed? | Manufacturing, logistics, acceptance | [Trust Anchors & Device Identity](../technology-options/trust-anchors-device-identity.md) |
| Provenance record | Where did this come from? | Sourcing, manufacturing, logistics, repair, transfer | [Product Acceptance](../practices-controls/product-acceptance.md) |
| SBOM or xBOM | What components are present? | Build, release, acceptance, update, vulnerability response | [Transparency Artefacts & Component Visibility](../technology-options/transparency-artefacts-component-visibility.md) |
| Reference measurement | What expected state should be compared? | Manufacturing, acceptance, deployment, update | [Attestation & Measured State](../technology-options/attestation-measured-state.md) |
| Attestation result | Is this in expected state now? | Acceptance, deployment, monitoring, repair | [Attestation & Measured State](../technology-options/attestation-measured-state.md) |
| Update record | Was an update authorized and installed? | Update, operations, repair | [Update & Recovery Mechanisms](../technology-options/update-recovery-mechanisms.md) |
| Vulnerability record | Are known exposures handled? | Sourcing, acceptance, deployment, update, audit | [Software, Component & Vulnerability Management](../practices-controls/software-component-vulnerability-management.md) |
| Repair record | Was trust restored after repair? | Repair, transfer, audit | [Evidence Repositories, Logs & Retention](../technology-options/evidence-repositories-logs-retention.md) |
| Decommissioning record | Was the asset retired safely? | Decommissioning, audit | [Audit & Compliance Readiness](../practices-controls/audit-compliance-readiness.md) |

## Use This Map

Use this map when drafting supplier requests, acceptance criteria, audit evidence lists, or implementation plans.
